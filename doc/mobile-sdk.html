越来越多的组织使用移动App,但是App程序的安全性，需要在多个层面来来解决。一个应用程序的每个部件都有其潜在的安全风险。避开这些威胁是一个消耗时间和资源的工作。

Mobile SDK为您的移动端程序提供了简单、直接的安全能力。
支持iOS和Android。

安全与实践

通讯层

敏感信息从服务器交换到客户端应用程序需要额外的安全层，仅仅依托主流技术如HTTPS可能不够，并可能引入一个额外的外部风险。
Mobile SDK提供了一个安全的信道，可以加密任何数据（文字，照片，QR码，等等）。提供端到端安全流程。

地理服务

平台可以确定的最终用户的位置，可以使用移动应用程序时，提供了标准化的访问基于位置的功能，独立于目标平台。
自非授权区域的身份验证、业务请求将不会被接受。

生命周期管理

一个客户端应用程序的最终用户的设备上运行。需要明确的知道在应用程序的认可和信任设备上运行。服务和设备之间的链路由产品部署的所有关键步骤进行管理。

密钥提供

密钥配置应该是灵活的，高度安全的。系统提供了全方位的提供机制，从手动激活选项、在线激活、QR码扫描、短信通知等，为基础的安全流程提供不同程度的便利性和安全性。

重新激活

任何零售应用程序需要的地方就生命周期管理为最终用户会经常改变他们的设备的时间过程。重新激活注册用户的过程中应运行尽可能顺利。

安全存储

提供安全的存储功能，允许所有应用程序数据，独立于任何操作系统或设备的简单加密。
该功能可与该装置结合能力组合使用，确保安全的存储被链接到特定的设备。

功能

  • 支持多种认证方式:提供一次性密码、数字签名、口令等。
  • 可定制的通知及响应
  • 设备绑定
  • 安全存储
  • 整合的注册管理流程
  • 开放服务平台

优势

  • 对最终用户透明
  • 多种安全体验:无需密码
  • 业务程序一个SDK实现
  • 后端接入方便
  • 支持PC等设备安全
  • 扩展支持硬件设备如IC卡
